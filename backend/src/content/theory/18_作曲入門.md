## 1. 從和弦進行開始

### 為什麼先寫和弦？

```
先寫旋律 → 再配和弦 = 需要更高的樂理功力
先寫和弦 → 再即興旋律 = 框架已定，更容易「湊」出旋律

對初學者來說，和弦進行是最好的「跳板」。
```

### 起步方法

```
Step 1: 選一個常見進行
  例：vi - IV - I - V（Am - F - C - G）

Step 2: 設定 BPM 和拍號
  例：BPM 120, 4/4 拍

Step 3: 用吉他把和弦彈出來，感受它的情緒
  → 聽起來像什麼？快樂？憂傷？有力？

Step 4: 在和弦的基礎上哼旋律
  不要想太多，嘴巴自然哼出來的就是旋律
  用手機錄下來！

Step 5: 把哼的旋律用吉他找出音，記下來
```

## 2. 旋律發展手法

### 反覆（Repetition）

```
一個動機重複出現：
  旋律 A → A → A → A

太多重複會無聊 → 加入微小的變化：
  旋律 A → A → A' → A   （A' = 稍微改變結尾）
```

### 模進（Sequence）

```
同一個旋律片段，在不同的音高上重複：

  旋律：C-D-E（在 C 上）
  模進：D-E-F（移高一個音重複同樣的形狀）
  再模進：E-F-G

聽起來像「在爬樓梯」—— 能量一層一層往上。
```

### 對比（Contrast）

```
用完全不同的旋律創造新鮮感：

  Verse：旋律 A（低沉、節制）
  Chorus：旋律 B（高昂、釋放）

A 和 B 的反差越大 → Chorus 的衝擊力越強。
```

## 3. Riff 創作

### Riff = 音階 + 節奏

```
Step 1: 選一個音階（如 Am Pentatonic）
Step 2: 用 3～8 個音組成一個短句
Step 3: 給它一個有特色的節奏
Step 4: 重複、重複、重複

例：
  音：A - C - D - E（Am Penta 的前 4 個音）
  節奏：♩ ♩ ♪♪ ♩ （四分-四分-八分八分-四分）

  TAB：
  E|--5---8---5-7---5--|
      ♩   ♩   ♪♪  ♩

  加上 Palm Mute、Distortion → 就是一個搖滾 Riff。
```

### 讓 Riff 更有特色

```
1. 加入 Blue Note（♭5）
2. 加入推弦或滑音
3. 用切分節奏（重音移到反拍）
4. 加入空白（不彈的拍子也是 Riff 的一部分）
5. 改變一個音的八度位置（低八度→高八度 = 驚喜感）
```

## 4. 曲式結構設計

```
最經典的搖滾/流行曲式：

Intro(4) → Verse(8) → Pre-Chorus(4) → Chorus(8)
         → Verse(8) → Pre-Chorus(4) → Chorus(8)
         → Bridge(8)
         → Chorus(8)(可能升 Key)
         → Outro(4)

每個段落的角色：
  Intro = 第一印象（Riff / 氣氛）
  Verse = 敘事（低能量，鋪墊）
  Pre-Chorus = 蓄力（能量上升）
  Chorus = 爆發（最高能量，最好記的旋律）
  Bridge = 轉折（新的視角、新的和弦進行）
  Outro = 收尾（回歸 Intro / 漸弱 / 突然結束）
```

### 段落之間的能量曲線

```
能量
  │
  │         ★ Chorus    ★ Chorus        ★★ Final Chorus
  │        ╱╲          ╱╲              ╱  ╲
  │  Pre ╱    ╲  Pre ╱    ╲  Bridge ╱    ╲ Outro
  │    ╱        ╲  ╱        ╲    ╱        ╲
  │  ╱ Verse    ╲╱ Verse     ╲╱             ╲
  │╱                                          ╲
  └───────────────────────────────────────────── 時間
  Intro

整首歌像波浪：低→高→低→高→更高→結束
```

## 5. 從分析到創作

```
最有效的學習路徑：

1. 分析 10 首你喜歡的歌（用 16 篇的方法）
2. 找出它們共用的和弦進行模式
3. 用你找到的模式作為起點
4. 在上面寫你自己的旋律和 Riff
5. 安排曲式結構
6. 錄下來 → 聽 → 修改 → 再錄

你的第一首歌不會完美 → 寫 10 首之後回頭看第 1 首會覺得很爛 → 這就是成長。
```
